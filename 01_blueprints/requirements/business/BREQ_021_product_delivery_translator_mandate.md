# BREQ_021: Product Delivery Translator Mandate (Agent 8)

## 1. Core Objective & Business Niche
The primary purpose of **Agent 8 (The Product Delivery Translator)** is to continuously translate complex engineering outputs into clear, stakeholder-focused product documentation. It acts as the bilingual bridge between technical codebase changes and product/client expectations, ensuring that internal changelogs, customer-facing release reports, and user manuals stay accurate without administrative friction.

## 2. Topological Git-Tree Execution Matrix
Agent 8 operates entirely branch-name agnostically. It determines its documentation compilation scale and granularity by analyzing the mathematical topological distance between the Git Source Node and Target Node during a Pull Request lifecycle event:

* **Atomic Merges (Micro-Scale):** Triggered on isolated, low-level branch merges (e.g., sub-task into feature). Output focuses purely on localized component mutations, immediate code comment extractions, and technical continuity.
* **Composite Merges (Macro-Scale):** Triggered when an intermediate branch aggregates multiple nested child branches (e.g., feature trees into intermediate epics, staging, or dev branches). Agent 8 recursively parses all previous micro-logs in the Git history tree, deduplicates overlapping mutations, and builds a consolidated system-impact summary.
* **Global Merges (System-Scale):** Triggered upon root product delivery deployments (e.g., pre-prod or production milestones). Compiles the accumulated macro-history into consumer-grade, non-technical Client Release Notes and User Manual delta updates.

## 3. Strict Boundary Lines & Operational Constraints
* **Absolute Client Quarantine (No Direct Delivery):** Agent 8 is strictly forbidden from publishing, emailing, or pushing release notes directly to external clients or production web portals. It functions purely as a text generator.
* **Absolute Ingestion Limit:** Agent 8 cannot parse raw application source code directories. To guarantee data stream separation, it reads exclusively from the structured files emitted by Agent 7 inside `/governance/technical/`.
* **Automated Transient Pipeline State:** Agent 8 operates autonomously to completion without human-in-the-loop dependencies at its boundary. All generated documentation across all scales must be written to the staging directory `/governance/delivery/staging/` with an initial state of `status: PENDING_COMPLIANCE`. It cannot bypass the downstream validation gatekeeper.

## 4. Business Output Requirements
All artifacts generated by Agent 8 must be structured using the following criteria:
* **The Compliance Tracking Header:** Every file must header-bind itself to the active milestone and the ancestral technical documents it is translating.
* **Jargon Separation:** Product-scale logs must separate technical infrastructure modifications (e.g., database optimizations) from user-facing functional modifications (e.g., feature additions), ensuring target audiences receive clean text.
